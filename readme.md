# Dropsy Airdrop Program

> Building a more transparent and community-driven airdrop infrastructure for Solana.

## Why Open Source?

Rather than waiting until the product is "finished," we want developers and security researchers to be able to review the architecture, identify issues, suggest improvements, and help shape the project as it evolves.

Our goals are simple:

- Enable public code review and community auditing.
- Build trust through transparency.
- Encourage collaborative development from day one.
- Create infrastructure that anyone can inspect, learn from, and improve.

Open source doesn't make software secure by itself, but it creates an environment where security can continuously improve through community review.

---

## Current Status

Dropsy is currently in active development.

The repository should be considered **early-stage** and **under active iteration**. APIs, architecture, and features may change as we gather feedback and improve the protocol.

We welcome constructive feedback, bug reports, and design suggestions throughout this process.

## Security

Security is one of the primary reasons this repository is public.

If you discover a vulnerability or believe you've found a security issue, please report it responsibly instead of publicly disclosing it immediately.

Responsible disclosure helps protect future users while giving us time to investigate and resolve the issue.

## Our Vision

Our goal isn't simply to build another airdrop tool.

We're working toward an open, community-driven infrastructure where projects can distribute tokens more transparently, contributors can participate meaningfully, and developers can build on top of an ecosystem they trust.

Every contribution, whether it's code, documentation, testing, security research, or thoughtful feedback that helps move Dropsy forward.
